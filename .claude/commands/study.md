이 프로젝트는 PQC 세미나 학습용이다. 사용자가 지정한 강의 회차 또는 주제(`$ARGUMENTS`, 예: "3강 LWE" 또는 "Ring-LWE")에 대해 아래 순서로 진행한다.

## 1. 자료 대조

`Lecture slides/`에서 해당 회차 슬라이드를 찾고, `Papers/A Gentle Introduction to Lattice-Based Cryptography.pdf`에서 같은 내용을 다루는 챕터/섹션을 대조해서 함께 참조한다. `Notes/`에 기존 정리본이 있으면 먼저 읽고 이어서 보강한다. 슬라이드와 교재 내용이 다르면 교재를 우선하고 차이를 알려준다.

## 2. 개념 설명 (수식 우선)

수식이 나오는 부분은 다음을 전부 짚어준다. 대충 넘어가지 않는다.

- **정의를 정확한 표기로 먼저 제시**: 집합, 차원, modulus 등을 명시 (예: $A \in \mathbb{Z}_q^{n \times m}$, $s \in \mathbb{Z}_q^n$, $e \leftarrow \chi^m$ 처럼 각 변수의 정의역과 분포까지)
- **왜 이렇게 정의되는지 직관**: 수식만 나열하지 말고 그 정의가 나온 이유(어떤 문제를 풀기 위함인지)를 설명
- **유도 과정**: 슬라이드/교재에 증명이나 유도가 있으면 생략된 중간 단계까지 채워서 보여준다. "자명하다"로 넘어가는 부분도 실제로 전개해서 보여준다
- **작은 숫자 예제**: 가능하면 작은 차원(n=2~4 등 구체적 수)으로 직접 계산한 예시를 만들어서 수식이 실제로 어떻게 작동하는지 보여준다
- **파라미터 간 관계**: 왜 특정 파라미터 크기(q, n, m, 오차 분포의 표준편차 등)를 선택하는지, 보안성/정확성과 어떤 트레이드오프가 있는지 설명
- **차원·타입 체크**: 행렬곱, modulus 연산 등에서 차원이 맞는지 매번 확인해서 짚어준다 (사용자가 헷갈리기 쉬운 지점)

## 3. 강의 간 연결

이전 회차와의 연결점을 짚어준다 (예: LWE라면 SIS와의 쌍대성, Ring-LWE라면 LWE의 대수적 일반화이고 어떤 수식이 어떻게 바뀌는지 구체적으로 비교).

## 4. 이해도 확인

설명이 끝나면 확인 질문 3~4개를 낸다. 최소 1개는 반드시 직접 손으로 계산해야 하는 문제로 낸다 (예: 주어진 작은 행렬/벡터로 SIS 해 찾기, LWE 샘플 직접 생성해보기). 나머지는 개념 이해와 응용 순서로.

## 5. 채점 및 재설명

사용자 답변, 특히 계산 문제는 중간 단계까지 검산한다. 틀리면 어느 단계에서 계산이 어긋났는지 짚어주고, 필요하면 더 쉬운 숫자로 다시 풀어본다.

## 6. 용어/기호 정리

새로 나온 용어나 표기법이 있으면 `Glossary.md`에 추가할지 물어보고, 동의하면 추가한다 (정의 + 기호 표기 + 어느 강의에서 나왔는지 함께 기록).

## 7. 저장

세션 끝에 `Notes/<회차 번호>. <제목>.md`에 오늘 학습한 내용을 정리해서 저장한다. 핵심 수식, 유도 과정 요약, 숫자 예제, 틀렸던 문제와 정답을 포함한다. 기존 파일 있으면 이어붙인다 (사용자 확인 후).

목표는 진도를 빨리 빼는 게 아니라 수식을 스스로 전개할 수 있을 정도로 이해시키는 것. 사용자가 막히면 더 쉬운 비유나 더 작은 숫자 예제로 다시 설명한다.
